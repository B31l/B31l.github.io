# 1. WSL 2 & Ubuntu
- 📌 [이전 버전 WSL의 수동 설치 단계](https://docs.microsoft.com/ko-kr/windows/wsl/install-manual)

## 1.1 기본 프로그램 설치
Microsoft Store에서 [Windows Terminal](https://apps.microsoft.com/store/detail/windows-terminal/9N0DX20HK701?hl=ko-kr&gl=KR) 및 [Ubuntu 20.04.4 LTS](https://apps.microsoft.com/store/detail/ubuntu-20044-lts/9MTTCL66CPXJ?hl=ko-kr&gl=KR)를 설치한다.
**Windows Terminal** 없이도 텐서플로우 작업환경을 구성하는 데는 문제 없지만, 많은 편리한 기능을 제공하고 커스터마이징이 자유롭기 때문에 설치한다.
**Ubuntu 20.04.4 LTS** 설치가 끝나면 실행한 다음 **username** 및 **password** 설정을 진행한다.

## 1.2 WSL 설치
PowerShell을 관리자 권한으로 실행한 다음, **Linux용 Windows 하위 시스템 사용 옵션** 기능을 사용하도록 설정한다. 
```shell
> dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart 
```

## 1.3 WSL 2로 업데이트 및 기본 버전으로 설정
윈도우 빌드가 18362보다 낮으면 WSL 2를 지원하지 않으므로 버전업을 진행하기 전에 먼저 가능 여부를 확인해야 한다. 
윈도우 빌드는 실행 창(`Windows 로고 키 + R`)에 **winver**을 입력하거나, 설정에서 **PC 정보**를 검색해서 확인할 수 있다.
업데이트가 가능하다면 PowerShell을 관리자 권한으로 실행한 다음, **Virtual Machine** 기능을 사용하도록 설정한다.
```shell
> dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
```
다음 명령어를 입력해 WSL2를 기본 버전으로 설정한다.
```shell
> wsl --set-default-version 2
```
위 방법이 작동하지 않으면 [이곳](aka.ms/wsl2kernel)에서 최신 WSL2 Linux 커널 업데이트 패키지를 설치한 뒤, 다음 명령어을 대신 입력한다.
```shell
> wsl --set-default-version Ubuntu-20.04 2
```
WSL 버전은 다음과 같이 확인할 수 있다.
```shell
> wsl -l -v

  NAME                   STATE           VERSION
* Ubuntu-20.04           Running         2
```
VERSION이 2로 표시되는 것을 확인할 수 있다.

## 2.2 Ubuntu에 Docker 엔진 설치
- 📌 [Ubuntu에 Docker 엔진 설치](https://docs.docker.com/engine/install/ubuntu/)

먼저 Ubuntu를 실행한다.
HTTPS를 통해 Reopsitory를 사용할 수 있도록 apt를 업데이트하고 패키지를 설치한다.
``` shell
$ sudo apt-get update
$ sudo apt-get install \
    ca-certificates \
    curl \
    gnupg \
    lsb-release
```
서명을 위한 Docker의 공식 GPG 키를 추가한다.
```shell
$ curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
```
리포지토리를 설정한다.
```shell
$ echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
```
apt를 업데이트하고 Docker 엔진을 설치한다.
만약 특정 버전을 설치하고 싶다면 [이곳](https://docs.docker.com/engine/install/ubuntu/#install-docker-engine)의 두 번째 단락을 참고한다.
``` shell
$ sudo apt-get update
$ sudo apt-get install docker-ce docker-ce-cli containerd.io docker-compose-plugin
```
hello-world 이미지를 실행해 Docker 엔진이 올바르게 설치되었는지 확인한다.
```shell
$ sudo docker run hello-world
```

## 2.3 설치 후 단계 (Optional)
- 📌 [선택적 설치 후 단계](https://docs.docker.com/engine/install/linux-postinstall/)

루트가 아닌 사용자 그룹에 root와 동일한 권한을 부여하면 Docker를 관리할 수 있다.
먼저, docker 그룹을 생성한다.
```shell
$ sudo groupadd docker
```
생성한 docker 그룹에 사용자를 추가한 다음, 재시작한다.
```shell
$ sudo usermod -aG docker $USER
$ newgrp docker 
```
`/var/run/docker.sock` 파일의 권한을 666으로 설정하여 그룹 내 다른 사용자도 접근 가능하게 변경한다.
```shell
$ sudo chmod 666 /var/run/docker.sock
```
이제 `sudo` 명령어 없이도 docker 명령어를 수행할 수 있는 것을 확인할 수 있다.
```shell
$ docker run hello-world
```

## 2.4 NVIDIA Docker 지원 설치
- 📌 [NVIDIA Docker 지원을 설치](https://docs.nvidia.com/datacenter/cloud-native/container-toolkit/install-guide.html#docker)

Linux 환경에서 Tensorflow가 GPU를 지원하려면 NVIDIA Docker 지원을 설치해야 한다.
NVIDIA 패키지 리포지토리 및 GPC 키를 설치한다.
```shell
$ distribution=$(. /etc/os-release;echo $ID$VERSION_ID) \
      && curl -fsSL https://nvidia.github.io/libnvidia-container/gpgkey | sudo gpg --dearmor -o /usr/share/keyrings/nvidia-container-toolkit-keyring.gpg \
      && curl -s -L https://nvidia.github.io/libnvidia-container/$distribution/libnvidia-container.list | \
            sed 's#deb https://#deb [signed-by=/usr/share/keyrings/nvidia-container-toolkit-keyring.gpg] https://#g' | \
            sudo tee /etc/apt/sources.list.d/nvidia-container-toolkit.list
```
apt를 업데이트하고 `nvidia-docker2` 패키지를 설치한 다음, 도커 데몬을 재시작해 설치를 완료한다.
```shell
$ sudo apt-get update
$ sudo apt-get install -y nvidia-docker2
$ sudo systemctl restart docker
```
이 시점에서 CUDA 컨테이너를 실행해 작동 성능을 테스트할 수 있다.

## 2.5 Tensorflow Docker 이미지 다운로드
- 📌 [Tensorflow Docker 이미지 다운로드](https://www.tensorflow.org/install/docker?hl=ko#download_a_tensorflow_docker_image)

공식 Tensorflow Docker 이미지는 [tensorflow/tensorflow](https://hub.docker.com/r/tensorflow/tensorflow/) Docker hub 저장소에 있으며, 다음과 같은 형식으로 리눅스에서 이미지를 태그할 수 있다.
베이스 태그를 사용해 Tensorflow 이미지의 버전을 지정할 수 있다.

| 베이스 태그 | 설명                                               |
| ----------- | -------------------------------------------------- |
| `latest`    | TensorFlow CPU 바이너리 이미지의 최신 출시(기본값) |
| `version`   | TensorFlow 바이너리 이미지의 버전 지정(예: 2.1.0)  |

각 베이스 태그에는 기능 추가를 위해 태그 변이를 적용할 수 있으며, 여러 변이를 함께 사용할 수도 있다.
| 태그 변이     | 설명                  |
| ------------- | --------------------- |
| `tag-gpu`     | GPU 지원              |
| `tag-jupyter` | Jupyter Notebook 포함 |

GPU를 지원하는 Jupyter Notebook 환경에서 최신 버전의 Tensorflow를 사용할 것이므로 다음과 같은 형식으로 이미지를 다운로드한다.
``` shell
$ docker pull tensorflow/tensorflow:latest-gpu-jupyter
```

## 2.6 Jupyter Notebook 실행
도커를 이용하기 위해서는 **Docker Daemon**이 실행중이여야 하므로 활성화한다.
```shell
$ sudo service docker start
```
다음 명령으로 컨테이너를 실행하고 생성된 링크로 이동하면 주피터 노트북을 사용할 수 있다.
```shell
docker run --gpus all -lt -v $(realpath ~/notebooks):/tf/notebooks -p 8888:8888 tensorflow/tensorflow:latest-gpu-jupyter
```
테스트를 위해 notebooks 폴더로 이동해 새 ipynb 파일을 생성한다.
다음 코드는 Tensorflow의 버전을 확인한다.
```python
import tensorflow as tf

print(tf.__version__)
```
다음 코드는 Tensorflow가 GPU를 사용하는지 확인한다.
```python
from tensorflow.python.client import device_lib

print(device_lib.list_local_devices())
```
주피터 노트북을 종료하려면 우측 상단의 `Quit` 버튼을 클릭한다.
